Obstacles with setting up environment variables and aliases

Goal

Set custom environment variables and aliases of console commands for specific user.


Deal with environment variables

Put definitions of environment variables into .profile in home directory. For example:

# ~/.profile: executed by the command interpreter for login shells.
# This file is not read by bash(1), if ~/.bash_profile or ~/.bash_login
# exists.
...
####### Environment variables for miscellaneous software #######
# Oracle JDK
export JAVA_HOME=/usr/local/jdk1.8.0_40
export PATH=$JAVA_HOME/bin:$PATH

# Groovy
export GROOVY_HOME=$HOME/bin/groovy-2.4.1
export PATH=$GROOVY_HOME/bin:$PATH

Deal with aliases

Create .bashrc in home directory (if it doesn't exist) and put definitions of aliases into it. For example:

####### Custom aliases ########################################
# reset terminal
alias tclear='setterm -reset'
# long and human readable listing of a directory
alias ll='ls -alh'

Where are obstacles?

Putting definitions of environment variables into .bashrc enables them only in a shell. Thus if program was ran from terminal it can access environment variables. If program was ran from desktop shortcut it can not discover environment variables.

Aliases are used only from terminal. So they should be declared in .bashrc (or .bash_aliases depending on a distro). Putting aliases into .profile makes them unaccessible for bash commands.
